DEV-294 — BEFORE FIX

Handsontable v17.1.0 (latest stable, bug present)
Bug: When using NestedHeaders + HiddenColumns + CollapsibleColumns together, the header colspan is not correctly reduced for hidden columns inside a collapsible group. This causes the header row to be wider than the data body — every header to the right of a collapsible group that contains a hidden column is shifted one (or more) positions off its data column.
Setup: 7 columns of data. The "Group A" header spans columns 0–2 (colspan 3). Column 2 is hidden via HiddenColumns. CollapsibleColumns and ManualColumnMove are enabled.

Expected: "Group A" should show colspan 2 (one hidden column subtracted). Headers B1, C1, D1, E1 should align with their data columns.
Actual (buggy): "Group A" keeps colspan 3. Headers B1–E1 are shifted one column to the right — B1 sits over C's data, C1 over D's data, etc.
Also try:
  1. Click the collapse triangle on "Group A" — the collapsed state is wrong.
  2. Use right-click → Show columns to unhide column 2 — alignment gets worse.
