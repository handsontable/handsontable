BEFORE DEV-1840 – Weak cell-meta typing

Renderers, editors and validators typed their cellProperties parameter as Record<string, unknown> or a locally-declared shadow type instead of the canonical CellProperties interface from src/settings.ts. This page shows the pre-fix state (Handsontable 17.0 release).

Problems in the pre-fix code

  • 16 renderers (e.g. htmlRenderer, intlTimeRenderer, intlDateRenderer, checkboxRenderer, …) accepted Record<string, unknown> — so IDE completion showed no known properties.
  • 9 editors + editorManager.ts used Record<string, unknown> or local shadow interfaces, breaking the link to the canonical API.
  • 7 validators typed this as a local CellMeta = Record<string, unknown> & { allowEmpty?: boolean } instead of canonical CellProperties.
  • numericType/valueSetter.ts used a bespoke NumericCellMeta type.
  • intl-* renderers/validators re-declared their own type CellProperties = Record<string, unknown> & { … } local shadow types that drifted from the real interface.

Example: timeRenderer.ts — BEFORE

// ❌ BEFORE: local shadow type — unknown at the IDE level
type CellProperties = Record<string, unknown> & {
  timeFormat?: Intl.DateTimeFormatOptions;
  locale?: string;
  allowEmpty?: boolean;
  instance?: object;
};

export function timeRenderer(
  hotInstance: Record<string, unknown>,
  TD: HTMLTableCellElement,
  …
  cellProperties: CellProperties  // ← shadow, not canonical
): void { … }

Example: timeValidator.ts — BEFORE

// ❌ BEFORE: local CellMeta shadow type
type CellMeta = Record<string, unknown> & { allowEmpty?: boolean };

export function timeValidator(
  this: CellMeta,   // ← weak: IDE sees only allowEmpty, not the full CellProperties
  value: unknown,
  callback: (valid: boolean) => void
): void { … }
