DEV-1858: Page Up/Down with frozen rows — viewport scroll bug

Version: Before Fix v17.1.0 CDN After Fix PR Build
🐛 Bug: When Page Up moves selection into the top frozen rows, the master viewport stays scrolled — row headers appear misaligned with the frozen overlay.
How to reproduce the bug:
1. Scroll the table down so rows 10–20 are visible.
2. Click any cell in the visible area (e.g. row 15, column A).
3. Press Page Up repeatedly until selection moves into the top frozen rows (rows 1–2).
4. Expected: Master viewport scrolls back to top, row headers align with frozen rows.
5. Actual: Master viewport stays scrolled — row headers show wrong numbers under frozen header rows.
