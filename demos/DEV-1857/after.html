DEV-1857: Middle mouse button autoscroll

Issue: #2722 — fix: middle-click mousedown no longer calls preventDefault()

Switch to: BEFORE FIX v17.1.0 AFTER FIX patched
Fix: The rootElement mousedown handler now returns early for middle-button clicks (button 1) via a new isMiddleClick() helper. The default action is preserved, allowing the browser to start its native autoscroll panning.

How to test (Windows / Linux):
  1. Middle-click (press the scroll wheel) on any grid cell below.
  2. The panning cursor (⊕) appears. Move the mouse to scroll the grid.
  3. The status panel shows defaultPrevented: false.
  4. Left-click still shows defaultPrevented: true — text selection guard is unaffected.
Note: This demo loads Handsontable v17.1.0 from CDN and applies the equivalent of PR #12754 as a JavaScript patch: a capture-phase mousedown listener intercepts middle-clicks before HOT's handler and replaces event.preventDefault with a no-op, matching the merged fix behavior exactly.
