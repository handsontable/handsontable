DEV-1826 — Before Fix

❌ BROKEN

FalseCJS: single ESM-syntax index.d.ts typed both ESM and CJS entry points with no "type" field

← Before Fix (current) After Fix →

Problems Identified

🎭
H1 — FalseCJS A single index.d.ts typed both the ESM (.mjs) and CJS (.js) entry points. No "type" field in package.json. Under moduleResolution: "node16", TypeScript treated the ESM-syntax declarations as CJS — giving wrong type resolution to ESM consumers.
❗
H1 — FalseExportDefault Same root cause: ESM declaration used for both conditions; CJS consumers using import Handsontable from 'handsontable' got incorrect default export resolution.
📦
H1 — 5 Broken Exports (publint) Wildcard glob rules in prepare-package-for-publish.mjs had no nodir: true, so directory paths were emitted into the exports map (e.g., ./cellTypes/checkboxType/ instead of a file). publint flagged 5 errors + 1 warning.
🔍
H1 — Weak Export Validation Export validation used existsSync() which returns true for directories, allowing dangling directory-path exports to pass silently.
🚫
H2 — No CI Gate verify-emitted-types.yml ran tsc@5.1.6 to test the type floor but never ran publint or attw to check package resolution. Package export breakage could ship undetected.
