DEV-1864 — BEFORE FIX: NaN regression in getSyncCalculationLimit

The regression (intermediate broken state):
When replacing >> 0 with Math.trunc(), syncLimit could be undefined if not explicitly configured, causing Math.trunc(undefined) = NaN.

With a NaN limit, syncLimit >= 0 is false, so ALL columns are auto-sized asynchronously only — the grid first renders with narrow default widths, then jumps.
Buggy implementation (simulates the regression):
// BEFORE FIX — NaN regression getSyncCalculationLimit() { const settings = this.hot.getSettings().autoColumnSize; let limit = AutoColumnSize.SYNC_CALCULATION_LIMIT; const colsLimit = this.hot.countCols() - 1; if (settings && typeof settings === 'object') { const syncLimit = settings.syncLimit; if (typeof syncLimit === 'string' && syncLimit.endsWith('%')) { limit = Math.round(colsLimit * parseFloat(syncLimit) / 100); } else { limit = Math.trunc(syncLimit); // ← BUG: Math.trunc(undefined) = NaN } } return Math.min(limit, colsLimit); // Math.min(NaN, N) = NaN }
