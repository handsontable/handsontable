DEV-1864 — AFTER FIX: NaN guard in getSyncCalculationLimit

The fix (DEV-1864, PR #12761):
Replace Math.trunc(syncLimit) with an explicit Number.isFinite guard so that undefined, NaN, or any non-finite value safely falls back to 0.

The sync phase now runs correctly for finite values, and columns are auto-sized on first render without a visual width flash.
Fixed implementation:
// AFTER FIX — with NaN guard (S7767) getSyncCalculationLimit() { const settings = this.hot.getSettings().autoColumnSize; let limit = AutoColumnSize.SYNC_CALCULATION_LIMIT; const colsLimit = this.hot.countCols() - 1; if (settings && typeof settings === 'object') { const syncLimit = settings.syncLimit; if (typeof syncLimit === 'string' && syncLimit.endsWith('%')) { limit = Math.round(colsLimit * parseFloat(syncLimit) / 100); } else { // ✓ FIX: explicit NaN guard — undefined/NaN fall back to 0 const numericSyncLimit = Number(syncLimit); limit = Number.isFinite(numericSyncLimit) ? Math.trunc(numericSyncLimit) : 0; } } return Math.min(limit, colsLimit); }
