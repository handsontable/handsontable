DEV-1777: After Fix

AFTER FIX
What changed: The new isVisible() function uses element.checkVisibility() (Chrome 105+, Firefox 106+, Safari 17.4+) as a fast O(1) native check instead of an O(DOM depth) ancestor walk with getComputedStyle() calls on every node.

Additionally, alignOverlaysWithTrimmingContainer() in Walkontable now caches holder sizing behind a fingerprint and uses a ResizeObserver for invalidation, avoiding redundant cloneNode + getComputedStyle measurements on every scroll step.

Combined, these changes reduce Layout + StyleRecalc cost by ~23% (234.7ms → 181.6ms for a 40-step scroll), bringing the docs/blank-page overhead ratio from 1.99× down to 1.24×.
