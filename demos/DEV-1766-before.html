DEV-1766 — BEFORE fix (v17.0.0)

Bug: Calling updateSettings({ colWidths: undefined }) on a table with nestedHeaders enabled does NOT re-expand columns to fit cell content. Column 0 stays at the ghost-table floor (~28px) instead of expanding to content width (500+px).
Steps to reproduce:
  1. Table initialises with colWidths: 100 and nestedHeaders.
  2. Click Reset colWidths to call updateSettings({ colWidths: undefined }).
  3. Observe column 0 — it should expand to fit the long content, but it stays narrow (BUG).
