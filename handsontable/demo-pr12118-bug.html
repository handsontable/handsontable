PR #12118 - manualColumnResize / manualRowResize broken in CSS-scaled container

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (v17.0.1). When the Handsontable container has transform: scale(0.5), drag deltas for manualColumnResize and manualRowResize are not divided by the scale factor. Moving the mouse 25 visual pixels makes the column grow by only 25 layout pixels instead of 50. The resize handle is also misaligned relative to the column header boundary.
How to observe the bug:
  1. The grid below is rendered at 50% scale via transform: scale(0.5).
  2. Hover over the right edge of any column header — the resize handle appears.
  3. Notice the handle is mispositioned (not at the header edge).
  4. Drag the handle right to resize a column. Watch the width delta counter — it shows the change the plugin applied vs. what your mouse actually moved.
  5. Expected: mouse moved 40px → column grew 80px. Actual: column grew only 40px (wrong).
  6. Compare with the PR Fix page where the scale is accounted for.
Grid is displayed at scale(0.5). Each visual pixel = 2 layout pixels. Drag deltas should be divided by 0.5 (doubled) but the bug ignores the scale factor.
