PR #12152 - Sort/filter/dropdown indicators missing on rowspanned nested headers

Build: Released v17.0.1 BUG PR #12129 Fix FIXED PR #12152 Fix FIXED
BUG present (v17.0.1). Even after #12129 adds rowspan support, plugins still used level === lastHeaderRow index comparisons to decide whether to inject sort arrows, dropdown buttons, and filter-active badges. A rowspanned header cell has a lower level index, so all these guards fail — the header gets no indicators and clicking it does not trigger sorting. This page uses v17.0.1 where rowspan is also not rendered, demonstrating the root cause: the "last level" check.
How to observe the bug:
  1. This 2-level header setup has an ID column that should span both header rows (as a rowspanned header that reaches the bottom).
  2. In v17.0.1, the sort arrow and dropdown button are absent from the top-row header — the level guard (level !== lastRow) rejects it.
  3. Compare with the Fix page where the rowspanned ID header has all indicators.
