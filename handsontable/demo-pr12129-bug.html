PR #12129 - nestedHeaders: rowspan property is silently ignored

Build: Released v17.0.1 BUG PR #12129 Fix FIXED PR #12152 Fix FIXED
BUG present (v17.0.1). The nestedHeaders plugin ignores the rowspan property on header config objects. A header declared as { label: 'ID', rowspan: 2 } renders as a single-row cell — no rowspan attribute is set on the <th>. All sort indicators and dropdown buttons are missing from such headers.
How to observe the bug:
  1. The grid is configured with two header rows. Column 0 uses { label: 'ID', rowspan: 2 } — it should span both header rows.
  2. In v17.0.1 the ID header occupies only the top row, leaving an empty gap below it.
  3. The rowspan attribute on the TH is 1 (not 2).
  4. Compare with the Fix page where ID spans both rows correctly.
