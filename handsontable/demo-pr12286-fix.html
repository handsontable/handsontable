PR #12286 - columnHeaderHeight now synced to actual DOM height in all overlays

Build: Released v17.0.1 BUG PR Fix FIXED
FIX applied (PR #12286 — local build). A new syncOversizedColumnHeadersWithDOM() method runs after the master table calls adjustColumnHeaderHeights(). It re-measures the actual rendered THEAD row heights from the DOM and updates oversizedColumnHeaders to match. When overlays then call adjustColumnHeaderHeights(), they receive the real master height — not the columnHeaderHeight setting — and all THEADs stay synchronized. 
// colHeader.js — NEW method (simplified):
syncOversizedColumnHeadersWithDOM() {
  const masterThead = this.hot.rootElement.querySelector('.ht_master thead tr');
  if (!masterThead) return;
  const domHeight = masterThead.getBoundingClientRect().height;
  const stored = this.hot.view._wt.wtSettings.getSetting('oversizedColumnHeaders', 0) ?? 0;
  if (Math.abs(domHeight - stored) > 1) {
    this.hot.view._wt.wtSettings.settings.oversizedColumnHeaders[0] = Math.round(domHeight);
  }
}
