DEV-1718 / #9513 — comments popup blocked when first row is sparse

Build: Released v17.0.1 BUG DEV-1718 Fix FIXED
BUG present (v17.0.1). When object data has a sparse first row (fewer keys than declared columns), comments renders corner markers on all cells but silently blocks the popup for any column index beyond the sparse row's key count.
  • Row 0 has only C0 and C1 — columns C2..C5 are absent.
  • Comment on (0, 0) and (0, 1): popup shows. ✓
  • Comment on (0, 3) (C3, not in first row): popup silently blocked. ✗
Root cause: Comments.show() checks countSourceCols() which returns deepObjectSize(data[0]) = 2. Column index 3 ≥ 2 → early return false. 
// BUG — Comments.show() in v17.0.1 (src/plugins/comments/comments.js):
show() {
  const { row, col } = this.#getRangeCoords();
  if (row < 0 || row > this.hot.countSourceRows() - 1
      || col < 0 || col > this.hot.countSourceCols() - 1) {
    return false;  // countSourceCols() = data[0] key count = 2
  }              // col 3 ≥ 2 → early return → popup never shown
}
data[0] = { C0: 'Alice', C1: 10 } ↑ only 2 keys — countSourceCols() returns 2 data[1] = { C0: 'Bob', C1: 20, C2: 30, C3: 40, C4: 50, C5: 60 } data[2] = { C0: 'Carol', C1: 15, C2: 25, C3: 35, C4: 45, C5: 55 } data[3] = { C0: 'Dave', C1: 11, C2: 22, C3: 33, C4: 44, C5: 55 }
How to observe the bug:
  1. Click "showAtCell(0, 3)" — returns false, no popup appears. ✗
  2. Click "showAtCell(0, 0)" — returns true, popup appears. ✓
  3. Hover over the red corner markers in row 0 — only C0 and C1 show a popup. ✗
  4. Hover over red corner markers in rows 1–3 — all columns work fine (those rows are not sparse). ✓
  5. Compare with the Fix page where all cells work. ✓
