PR #12483 - date validator infinite loop

Build: Released v17.0.1 BUG PR #12483 Fix FIXED
BUG present (v17.0.1). Pasting a numeric value (e.g. 50000) into a date cell configured with correctFormat: true and allowInvalid: false causes an infinite validation loop:

paste "50000" → validator calls correctFormat() → returns "Invalid date" → setDataAtCell("Invalid date") → afterChange → validator on "Invalid date" → allowInvalid: false reverts to "50000" → afterChange → validator on "50000" → ∞ 
// BUG — dateValidator.js (before fix):
if (isValidDate && !isValidFormat) {
  if (this.correctFormat === true) {
    const correctedValue = correctFormat(valueToValidate, this.dateFormat);
    // correctedValue may be "Invalid date" — but we call setDataAtCell anyway:
    this.instance.setDataAtCell(this.visualRow, this.visualCol, correctedValue, 'dateValidator');
    valid = true;  // marked valid even when correctedValue === "Invalid date"
  }
}
Safety guard enabled: This demo limits afterChange to 20 calls and then destroys the grid to prevent the browser tab from freezing. In a real app without a guard, this would lock the browser.
How to observe the bug:
  1. Click "Paste '50000' into date cell".
  2. BUG: afterChange fires 20+ times before the guard kills the grid.
  3. Compare with the Fix page where it fires at most 2 times.
