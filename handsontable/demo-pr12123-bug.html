PR #12123 - stretchH: 'last' collapses last column to 0px in narrow viewport

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (v17.0.1). With stretchH: 'last', when the viewport is narrower than the combined width of all other columns, StretchLastStrategy.calculate() used Math.max(viewportWidth - allColumnsWidth + lastColumnWidth, 0) — setting the last column to 0px when the result goes negative. The column becomes completely invisible. Drag the slider left until the last column disappears.
How to observe the bug:
  1. The table has 3 columns defined at 80px, 80px, 100px with stretchH: 'last'.
  2. Drag the container width slider left toward 140px.
  3. Watch the Column C width counter — it drops to 0px when the viewport is narrower than 160px (the sum of the first two columns). Column C becomes invisible.
  4. Compare with the PR Fix page where Column C stays at 100px.
