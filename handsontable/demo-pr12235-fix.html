PR #12235 - Fast scrollbar drag rendering (StickyScrollStrategy)

Build: Released v17.0.1 BUG PR Fix FIXED
StickyScrollStrategy active (PR #12235). When the user grabs the native scrollbar, the master spreader (and overlay clone spreaders) switch to position: sticky. The browser handles positioning natively — no JavaScript repositioning lag. On mouseup the scroll position is corrected and a final render restores normal layout without a visible snap.
New code in this build: stickyScrollStrategy.js (+372 lines) — detects scrollbar mousedown, activates sticky mode (applySpreaderStyles('sticky', top, left)), syncs offsets after each scroll-triggered render via syncOffsets(), restores position: relative on mouseup. Integrated into overlays.js at 4 call sites. Disabled on Safari.
How to verify the fix:
  1. Click and hold the vertical scrollbar thumb on the right side of the grid.
  2. Drag it rapidly up and down several times.
  3. Watch the Sticky Mode indicator — it should turn green during drag.
  4. Release the mouse — the indicator returns to inactive, rendering stays correct.
  5. No blank rows, no position jump on release.
