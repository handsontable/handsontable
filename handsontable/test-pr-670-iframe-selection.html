RELEASE-670 — Selection border inside iframe

Reproduces E2E spec: Selection > running in iframe > should set correct wtBorder top position of selected cell (src/selection/__tests__/general.spec.js:1805)

What is being tested: When Handsontable runs inside an <iframe>, selecting cell B2 (row 1, col 1) must place the .wtBorder.current selection border at top: 29px (= one default row height in ht-theme-main). The spec fails intermittently with Expected '1px' to equal '29px'.

Suspected cause: The iframe's theme <link> stylesheet loads asynchronously after doc.write(). If Handsontable initialises before the stylesheet applies, rows render at an un-themed height and the border gets placed at the wrong offset. The failure is timing-dependent (flaky).

How to use this page: Click Run once to execute one test iteration, or Run 10× to stress-test for flakiness. Each run rebuilds the iframe from scratch (same as the spec's beforeEach). Watch the log — any FAIL entry confirms the race condition.
