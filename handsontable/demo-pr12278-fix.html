PR #12278 — Wrappers skip init-only settings in updateSettings

Demo: Bug v17.0.1 Fix local build
Fix — PR #12278: Framework wrappers (React, Vue, Angular) now read hot.getSettings()._initOnlySettings before calling updateSettings(). Any setting whose key appears in that list is silently skipped. This means a prop change to renderAllRows (or layoutDirection, renderAllColumns, ariaTags) no longer crashes the wrapper — and every other setting in the same call is applied normally. Core Handsontable still throws for direct calls; the filtering happens exclusively inside the wrapper layer.
