DEV-1220: DataProvider Plugin — Before (Static Data)

View: Before static data After dataProvider
Scenario: Product inventory grid with 500 rows — all data loaded at once in the browser.
Pagination, sorting, and filtering are handled entirely in the frontend.
All 500 rows are in memory and in the DOM regardless of how many are visible.
Problems with this approach:
  • All 500 rows transferred over the network on every page load
  • Sorting triggers a full client-side re-sort of all data
  • No server validation — invalid data can be saved locally then fail on backend
  • Create/Update/Delete require custom fetch logic wired up manually per-project
  • No loading state or error recovery — if the network fails, the grid silently shows stale data
