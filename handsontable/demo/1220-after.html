DEV-1220: DataProvider Plugin — After (Server-Side Integration)

View: Before static data After dataProvider
Scenario: Same 500-row product inventory — but now only the current page (25 rows) is fetched from the "backend" on every page change, sort, or filter.
The dataProvider plugin wires up pagination, sorting, filtering, and CRUD automatically.
What the dataProvider plugin provides:
  • fetchRows called with { page, pageSize, sort, filters } — only current page transferred
  • Server-side sorting and filtering — backend decides the order
  • onRowsCreate / onRowsUpdate / onRowsRemove — CRUD sent to backend automatically
  • Built-in loading overlay (via dialog: true) and error toasts (via notification: true)
  • Refetch button in error toast retries the last failed request automatically
