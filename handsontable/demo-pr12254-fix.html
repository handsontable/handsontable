Build: Released v17.0.1 BUG PR #12254 Fix FIXED PR #12255 Fix FIXED
FIX applied (PR #12254 — local build). Both isEmptyRow and isEmptyCol now compare each cell value against the corresponding dataSchema default for that property. A cell value that equals its schema default (e.g. false === false for a checkbox column) is now considered empty. Spare rows filled with schema defaults are correctly identified as empty, so minSpareRows maintains exactly the configured number of spare rows. 
// metaSchema.js / core.js — isEmptyRow updated (simplified):
// Now compares cell value to dataSchema default for that key:
if (value === schemaDefault || isObjectEqual(value, schemaDefault)) {
  continue; // treat as empty
}
