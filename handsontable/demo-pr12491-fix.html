PR #12491 - Password cell type — hashRevealDelay option

Build: Released v17.0.1 BEFORE PR #12491 Fix AFTER
FIX applied (PR #12491). The hashRevealDelay option is now supported by type: 'password' cells. When set to a positive number (milliseconds), the editor switches from a native <input type="password"> to a <input type="text"> with manual masking:
  • The last typed character stays visible for hashRevealDelay ms, then is replaced by hashSymbol.
  • All previously typed characters are already masked — only the current character is revealed.
  • getValue() always returns the real unmasked string stored in #realValue.
  • Setting hashRevealDelay: 0 or omitting it restores immediate masking (legacy behavior).
// FIX — passwordEditor.js (key additions):
#realValue = '';

  if (this.cellProperties.hashRevealDelay > 0) {
    this.TEXTAREA.type = 'text';    // switch to text for manual masking
    this.#inRevealMode = true;
  }
  super.open();
}

  // mask all previous chars with hashSymbol
  // leave the last char visible for hashRevealDelay ms, then mask it too
}

getValue() {
  return this.#inRevealMode ? this.#realValue : super.getValue();
}
