PR #12489 / #10980 — columnSummary ignores HyperFormula calculated cells

Build: Released v17.0.1 BUG PR #12489 Fix FIXED
BUG present (v17.0.1). When both the formulas plugin (HyperFormula) and columnSummary are active, cells containing formulas (e.g. =A2+B2) are silently skipped from all summary calculations — sum, min, max, average, and count.
  • Rows with plain numbers contribute to the summary. ✓
  • Rows with =formula values are excluded from the sum. ✗
  • The summary cell shows a lower total than it should. ✗
Root cause: columnSummary used getSourceDataAtCell which returns the raw string "=A2+B2". parseFloat("=A2+B2") → NaN → cell skipped. 
// BUG — columnSummary plugin (v17.0.1):
getCellValue(visualRow, visualCol) {
  return this.hot.getSourceDataAtCell(...);
  // returns "=A2+B2" for formula cells
  // parseFloat("=A2+B2") → NaN → cell skipped in sum/min/max/average/count
}
How to observe the bug:
  1. The "Total" column contains formulas =A+B for each row.
  2. The columnSummary row at the bottom should show the sum of all Total values.
  3. BUG: it shows only the sum of the plain-number rows — formula rows are excluded. ✗
  4. Compare "Summary (grid)" with "Correct sum" below.
  5. Edit cell A2 or B2 to change a value — the summary does not update. ✗
