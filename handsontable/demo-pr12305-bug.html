PR #12305 - Shared HyperFormula engine + autoColumnSize/autoRowSize causes TypeError

Build: Released v17.0.1 BUG PR #12305 Fix FIXED
BUG present (v17.0.1). afterFormulasValuesUpdate is broadcast to every Handsontable instance sharing the same HyperFormula engine. AutoColumnSize and AutoRowSize process all change addresses without filtering by sheetId. When Sheet2 changes arrive in the Sheet1 handler, out-of-range row/column indices cause toVisualRow / toVisualColumn to return null, propagating a TypeError into GhostTable sampling.
How to observe the bug:
  1. Two grids share the same HyperFormula engine: Table 1 (1 row × 2 cols), Table 2 (2 rows × 1 col).
  2. Click "Edit Table 2 cell" — this triggers afterFormulasValuesUpdate on both instances.
  3. Table 1 receives Sheet2 changes with row index 1 which is out of range for its 1-row layout.
  4. A TypeError is thrown. The error is captured and shown below.
