PR #12123 - stretchH: 'last' respects minimum column width in narrow viewport

Build: Released v17.0.1 BUG PR Fix FIXED
FIX applied (PR #12123 — local build). StretchLastStrategy.calculate() now checks if the calculated width would be less than the column's defined base width. When the viewport is too narrow, it calls this.stretchedWidths.clear() and returns without applying any stretch — preserving the last column's minimum width instead of shrinking it to 0px. 
// strategies/last.js — CHANGED:
const lastColumnWidth = Math.max(viewportWidth - allColumnsWidth + lastColumnWidth, 0);
const lastColumnWidth = viewportWidth - allColumnsWidth + this.#lastColumnWidth;
if (lastColumnWidth < this.#lastColumnWidth) {
  this.stretchedWidths.clear(); // don't shrink below defined width
  return;
}
