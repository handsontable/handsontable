PR #12490 - dragToScroll object config TypeScript type

Build: Released v17.0.1 BUG PR #12490 Fix FIXED
BUG present (v17.0.1 types). The dragToScroll TypeScript type was declared as boolean only. The plugin has always accepted an object form at runtime — { interval?: { min?, max? }, rampDistance? } — to fine-tune scroll speed and ramp distance. Using the object form in a TypeScript project caused a compile error, forcing users to cast with as any or ignore the error.
Type declaration (v17.0.1 — BUG)
// dragToScroll.d.ts — BEFORE fix
export type Settings = boolean;
// ^ object config completely absent from the type
TypeScript error when using object config
// TypeScript project — compile error with v17.0.1 types:
new Handsontable(container, {
  dragToScroll: {         // TS2322: Type '{ interval: {...} }' is not assignable
    interval: { min: 20, max: 500 },  // to type 'boolean'
    rampDistance: 120,
  },
});
TypeScript error TS2322 (v17.0.1)
Type '{ interval: { min: number; max: number; }; rampDistance: number; }' is not assignable to type 'boolean'.
Object literal may only specify known properties, and 'interval' does not exist in type 'boolean'.
TypeScript error TS2322 (v17.0.1)
Type '{ rampDistance: number; }' is not assignable to type 'boolean'.
Note: At runtime the plugin accepted the object form all along — only the TypeScript declaration was wrong. The live grid below shows the object config works in plain JavaScript. Compare with the Fix page where the type correctly allows the object form.
