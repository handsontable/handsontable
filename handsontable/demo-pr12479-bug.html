PR #12479 - columnSorting with boolean values in type: 'text' column

Build: Released v17.0.1 BUG PR #12479 Fix FIXED
BUG present (v17.0.1). When a column contains boolean data (true/false) and is typed as type: 'text', clicking the column header to sort does nothing — the row order never changes.

Root cause: the default comparator calls isNaN(value). isNaN(true) === false (JS coerces booleans to numbers), so booleans enter the numeric path. But parseFloat(true) returns NaN (it stringifies first: "true" cannot parse as a float). NaN < NaN and NaN > NaN are both false, so every pair returns DO_NOT_SWAP and the column never reorders. 
// BUG — sortFunction/default.js comparator (before fix):
return function(value, nextValue) {
  // isNaN(true) === false → booleans pass this check
  if (isNaN(value) === false && isNaN(nextValue) === false) {
    value = parseFloat(value);      // parseFloat(true) → NaN (stringifies first)
    nextValue = parseFloat(nextValue);
  }
  // NaN < NaN === false, NaN > NaN === false → always DO_NOT_SWAP
};
How to observe the bug:
  1. Click "Sort "Active" ASC" or "Sort "Active" DESC".
  2. BUG: the column header shows a sort indicator but rows do not reorder.
  3. The "Active" column (boolean, type: 'text') should have all false rows first (ASC) or all true rows first (DESC).
  4. Click the "Active" header directly to toggle sort — same non-result.
  5. Compare with the Fix page.
