FIX applied (DEV-1560 / PR #12468). clearHooks() is no longer called in SelectEditor.close(). Scroll listeners persist for the editor's lifetime and are cleaned up only on destruction — mirroring the pattern used by TextEditor.
  • 1st open → scroll: editor follows cell. ✓
  • Close (Esc), reopen → scroll: editor still follows cell. ✓
  • Multiple cycles: correct every time. ✓
// FIX — selectEditor.js (PR #12468):
close() {
  this.selectWrapper.style.display = 'none';
  this.clearHooks();  // ← removed
  super.close();
}

// Hooks cleaned up only on destroy:
destroy() {
  this.clearHooks();
  super.destroy();
}
