DEV-1560 / #11365 — Select editor repositions correctly after close & reopen

Build: Released v17.0.1 BUG DEV-1560 Fix FIXED
FIX applied (DEV-1560 / PR #12468). clearHooks() is no longer called in SelectEditor.close(). Scroll listeners registered in init() persist for the editor's lifetime and are only removed on destruction — mirroring the pattern used by TextEditor. refreshDimensions() already short-circuits when the editor is not in EDITING state, so the persistent hooks are no-ops while closed.
  • First open → scroll: editor follows the cell. ✓
  • Close (Esc), reopen → scroll: editor still follows the cell. ✓
  • Multiple open/close cycles: correct every time. ✓
// FIX — selectEditor.js close() (PR #12468):
close() {
  this.#select.style.display = 'none';
  this.clearHooks();  // ← removed: was stripping scroll listeners permanently
  super.close();
}

// Hooks are now cleared only on destroy():
destroy() {
  this.clearHooks();
  super.destroy();
}
// refreshDimensions() short-circuits: if (state !== EDITING) return; → safe persistent hook
