PR #12203 - Autofill across hidden columns overwrites extra visible columns

Build: Released v17.0.1 BUG PR #12203 Fix FIXED
BUG present (v17.0.1). When dragging autofill across hidden columns with copyPasteEnabled: false, the visual vs raw column index tracking diverges. getInputValue() was called with visualColumn (skipping hidden cols) instead of the raw column index, causing the input pattern to repeat into columns beyond the drag target.
How to observe the bug:
  1. Grid has 6 columns (A–F). Columns B and C (indices 1 and 2) are hidden with copyPasteEnabled: false.
  2. Cell A1 contains 'A'.
  3. Click "Simulate autofill A1 → D1" — equivalent to dragging the fill handle from A1 to D1.
  4. Cell D1 gets 'A' as expected. But cells E1 and F1 also get 'A' — bug.
  5. Expected: only D1 = 'A'; E1 and F1 remain null.
D1 (col 3)
—
Expected: 'A' (autofill target)
E1 (col 4)
—
Expected: null — BUG: 'A'
F1 (col 5)
—
Expected: null — BUG: 'A'
