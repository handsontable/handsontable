PR #12482 - Numeric European locale dot-thousands correctly parsed

Build: Released v17.0.1 BUG PR #12482 Fix FIXED
FIX applied (PR #12482). Two new helpers — isDotThousandsGroupedInteger and isDotThousandsGroupedFloat — detect European dot-grouped inputs when decimalSeparator === ','. getParsedNumber and the numeric valueSetter now strip the dots and normalize the comma before calling parseFloat, so all three cases are stored correctly:
  • 7.000 → stored as 7000 ✓
  • 1.234.567 → stored as 1234567 ✓
  • 7.000,25 → stored as 7000.25 ✓
// FIX — helpers/number.js getParsedNumber (two new branches added):
if (isDotThousandsGroupedInteger(numericData, decimalSeparator)) {
  return parseFloat(normalizedNumericData.replace(/\./g, ''));
  // "7.000"     → remove dots → parseFloat("7000")     → 7000
  // "1.234.567" → remove dots → parseFloat("1234567")  → 1234567
}

if (isDotThousandsGroupedFloat(numericData, decimalSeparator)) {
  return parseFloat(normalizedNumericData.replace(/\./g, '').replace(',', '.'));
  // "7.000,25"  → remove dots, comma→dot → parseFloat("7000.25") → 7000.25
}
