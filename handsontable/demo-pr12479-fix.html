FIX applied (PR #12479). Boolean values in a type: 'text' column now sort correctly: false sorts before true ascending (and after descending). The fix converts booleans to numbers (false → 0, true → 1) at the start of the comparator, before the isNaN check, so they flow through the existing numeric sort path cleanly.
