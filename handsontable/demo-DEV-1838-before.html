DEV-1838: indexSyncer private module leaked via package exports

This page demonstrates the bug — the extglob pattern +(index)* exposes private internal modules.

Version: BEFORE fix BUG AFTER fix FIXED
🐛 Bug present: The ./plugins/*/+(index)* extglob pattern in package.json exports matches any filename starting with index, including the private indexSyncer/ helper directory inside plugins/formulas/. External consumers can import handsontable/plugins/formulas/indexSyncer — a private implementation detail.

Buggy pattern in package.json exports

"./plugins/*": {
  "allowedExports": [
    "./plugins/filters/condition/**",
    "./plugins/filters/logicalOperations/**"
  ],
  "export": "./plugins/*/+(index)*"   // ← extglob: matches anything STARTING with "index"
}

The extglob +(index)* means: one or more occurrences of "index" followed by anything. So it matches index.js, index.mjs — but also indexSyncer/, indexSyncer/index.js, etc.
