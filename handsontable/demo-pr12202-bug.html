PR #12202 - fixedColumnsStart: 2nd fixed column header 1px misalignment when selected

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (v17.0.1). With fixedColumnsStart: 2 in the ht-theme-main or ht-theme-horizon theme, selecting the second fixed column (B) reveals a 1px horizontal misalignment between the selected column header and its data cells. The root cause is an overly broad CSS rule in _base.scss that applied border adjustments to th:nth-child(2) without scoping it to the corner overlay.
How to observe the bug:
  1. Click "Select Column A" — both the header and column cells align correctly (column A is fine).
  2. Click "Select Column B" — the header content shifts 1px to the right relative to the cell content.
  3. Look at the Left-edge offset difference metric below — it shows a non-zero value for column B.
  4. Compare with the PR Fix page where both columns align perfectly.
