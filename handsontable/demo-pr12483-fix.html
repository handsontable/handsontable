PR #12483 - date validator no longer loops on uncorrectable values

Build: Released v17.0.1 BUG PR #12483 Fix FIXED
FIX applied (PR #12483). The date validator now checks whether correctFormat() produced a genuinely valid date before calling setDataAtCell. If the corrected value fails strict moment(corrected, dateFormat, true).isValid(), the cell is marked invalid immediately without writing back — breaking the loop. With allowInvalid: false, the cell reverts cleanly in at most 2 afterChange cycles. 
// FIX — dateValidator.js:
if (isValidDate && !isValidFormat) {
  if (this.correctFormat === true) {
    const correctedValue = correctFormat(valueToValidate, this.dateFormat);
    const isCorrectedValueValid = moment(correctedValue, this.dateFormat, true).isValid();

    if (isCorrectedValueValid) {
      this.instance.setDataAtCell(this.visualRow, this.visualCol, correctedValue, 'dateValidator');
    }
    valid = isCorrectedValueValid;  // false for "Invalid date" → no setDataAtCell → no loop
  }
}
