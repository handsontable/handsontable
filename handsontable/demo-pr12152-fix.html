FIX applied (PR #12152 — local build). A new isBottomMostColumnHeader(TH) helper replaces all level === lastHeaderRow index comparisons across the dropdownMenu, filters, and columnSorting plugins. It checks whether headerLevel + rowspan ≥ totalHeaderRows, so a rowspanned header that visually reaches the bottom row is correctly treated as sortable/filterable. The column sorting keyboard shortcut also relaxes its guard from row === -1 to row < 0. 
// element.js — new DOM helper (simplified):
export function isBottomMostColumnHeader(TH) {
  const rowspan = Number.parseInt(TH.getAttribute('rowspan'), 10) || 1;
  return headerLevel + rowspan >= headerRows.length;
}

// dropdownMenu.js / filters.js / columnSorting.js — guard replaced:
if (level !== headerRowList.length - 1) { return; }
if (!isBottomMostColumnHeader(TH)) { return; }
