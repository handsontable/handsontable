DEV-1940: Walkontable DOM renderer BEFORE fix

v17.1.0 — diffing renderer with coordinate-keyed NodesPool (unbounded memory growth)
View: Before fix v17.1.0 CDN After fix PR build
Bug: The old NodesPool maintained a Map<number, HTMLElement> keyed by source coordinate (row × 1 048 576 + col). Every unique (row, col) cell ever scrolled into view added a new entry — the map never shrinks. Scrolling through 1 000 rows × 20 cols creates up to 20 000 cached TD/TH nodes retained in memory indefinitely.

How to test: Click Scroll through all rows, then watch the Cached nodes (NodesPool) counter climb. After the full scroll, compare to the Live viewport nodes. The cached count far exceeds what is actually rendered — those are leaked DOM nodes. Switch to the After fix page: the cached count stays bounded at viewport size.
