PR #12307 - disableVisualSelection treating undefined as true

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (released v17.0.1). In Highlight.isEnabledFor(), the check disableHighlight === false uses strict equality, so undefined === false is false - selection gets disabled even though it should be enabled by default.

Steps to reproduce:
  1. Click any cell in the left grid (disableVisualSelection: undefined).
  2. Bug: no blue selection highlight appears - looks the same as the right grid (disabled).
  3. The middle grid (false) works correctly - click it to see the expected selection highlight.
disableVisualSelection: undefined BUG: selection disabled
Should behave like false (selection enabled). With the bug it behaves like true.
disableVisualSelection: false CORRECT: selection works
Explicit false - selection enabled. Reference baseline.
disableVisualSelection: true CORRECT: selection disabled
Explicit true - selection intentionally disabled. Reference baseline.
