PR #12290 - Space key in filter search clears the list

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (released v17.0.1). The filter search input passes the raw value (including spaces) to the filter logic. Pressing Space results in a query like " " or "New " which either matches nothing or breaks multi-word search.

Steps to reproduce:
  1. Click the dropdown arrow ▼ on the City column header to open the filter menu.
  2. Click inside the Search input field at the top of the "Filter by value" section.
  3. Press the Space key (or type just a space character).
  4. Bug A: the checkbox list empties completely — no city is visible.
  5. Alternatively: type New then press Space.
  6. Bug B: the list empties even though "New York" and "New Orleans" should still match.
Scenario A — type a single space Input: " "
Expected: all cities shown (empty query = no filter)
BUG: list is empty
Scenario B — type "New " (with trailing space) Input: "New "
Expected: "New York", "New Orleans" shown
BUG: list is empty
Scenario C — type "new" (no space) Input: "new"
Expected: "New York", "New Orleans" shown
WORKS: correct

Click the ▼ dropdown button on the City column to open the filter.
