PR #12143 - setSourceDataAtCell now correctly writes to nested child row

Build: Released v17.0.1 BUG PR Fix FIXED
FIX applied (PR #12143 — local build). DataSource.setAtCell() now resolves the row object via the modifyRowData hook before writing. The nestedRows plugin hooks into modifyRowData to return the correct nested child object for a given visual row index. Previously, the write went directly to this.data[row] — the flat source array — ignoring the nested structure. 
// dataMap/dataSource.js — CHANGED:
const dataRow = this.modifyRowData(row); // resolves to nested child via hook
if (!Number.isInteger(column)) {
  setProperty(this.data[row], column, value);
  setProperty(dataRow, column, value);
} else {
  this.data[row][column] = value;
  dataRow[column] = value;
}
