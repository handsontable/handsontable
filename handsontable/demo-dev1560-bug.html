DEV-1560 / #11365 — Select editor drifts after close & reopen

Build: Released v17.0.1 BUG DEV-1560 Fix FIXED
BUG present (v17.0.1). The select cell editor loses its scroll-tracking behavior after the first open/close cycle.
  • First open → scroll: editor follows the cell correctly. ✓
  • Close (Esc), reopen → scroll: editor stays frozen at its old screen position while the cell scrolls away. ✗
Root cause: SelectEditor.close() called clearHooks(), permanently removing the afterScrollVertically / afterScrollHorizontally listeners. Since registerHooks() runs only once in init(), the editor has no scroll hooks from the second open onward. 
// BUG — selectEditor.js close() (v17.0.1):
close() {
  this.#select.style.display = 'none';
  this.clearHooks();  // ← strips afterScrollVertically + afterScrollHorizontally permanently!
  super.close();
}
// registerHooks() only runs in init() — never called again → no scroll repositioning on reopen
How to trigger the bug — follow these steps in order:
  1. Double-click any cell in the "Status" column to open the select editor.
  2. Click "Scroll down (with editor open)" — editor should follow the cell. ✓
  3. Press Esc to close the editor (or click outside).
  4. Double-click the same cell again to reopen the editor.
  5. Click "Scroll down (with editor open)" again.
  6. BUG: editor stays frozen at its old position while the cell scrolls away. ✗
