PR #12211 - nestedHeaders + columnSorting no longer crashes with disableVisualSelection

Build: Released v17.0.1 BUG PR #12211 Fix FIXED
FIX applied (PR #12211 — local build). A single null-guard was added in NestedHeaders#onBeforeHighlightingColumnHeader. When disableVisualSelection: 'current' is active the focus highlight's visualCellRange is null; the handler now returns early in that case instead of attempting to read .highlight.col off null. 
// nestedHeaders.js — onBeforeHighlightingColumnHeader:
  const focusHighlight = this.hot.selection.highlight.getFocus();
  const focusVisualCellRange = focusHighlight.visualCellRange;

  if (focusVisualCellRange === null) {
    return; // disableVisualSelection:'current' leaves visualCellRange null
  }

  focusHighlight.visualCellRange.highlight.col = columnIndex;
  focusVisualCellRange.highlight.col = columnIndex;
