DEV-1025 — width option ignored when height is not set

Build: Released v17.0.1 BUG DEV-1025 Fix FIXED
BUG present (v17.0.1). Two broken cases when width is set without specifying height:
  • Case 1: { width: 300 } (no height) — width is ignored, grid expands to full container width. ✗
  • Case 2: { width: 300, height: undefined } — layout breaks: right part of the table is pushed out of view, horizontal scroll doesn't appear. ✗
  • Working control: { width: 300, height: 200 } — correctly constrained. ✓
// BUG — width ignored when height is omitted:
new Handsontable(el, {
  width: 300,
  // height: omitted → width constraint is not applied, grid fills container
});

// BUG — height: undefined breaks layout:
new Handsontable(el, {
  width: 300,
  height: undefined,
  // right part pushed out of view, no horizontal scrollbar
});
How to observe the bugs:
  1. The red line marks 300 px — the configured width.
  2. Case 1: grid expands past the red line — width: 300 has no effect. ✗
  3. Case 2: columns on the right are clipped with no horizontal scrollbar. ✗
  4. Case 3: correctly stops at the red line. ✓
  5. Compare with the Fix page where all cases work. ✓
