Undo after column remove with merged cells

Version: Released v17.1.0 — BUG PR Build fixed
Fix (PR #12801): Removing a column that overlaps a merged cell area and then pressing Ctrl+Z now correctly restores the merged cell range alongside the column data.

How to verify the fix:
1. The grid loads with several merge configurations (see labels below).
2. Remove a column using the buttons or right-click context menu.
3. Press Ctrl+Z (or click Undo).
4. FIXED: the merged cell is restored to its original row/col/rowspan/colspan.
5. Press Ctrl+Y (Redo) — the merge shrinks or disappears again as expected.
