PR #12143 - setSourceDataAtCell updates parent row instead of nested child when nestedRows is enabled

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (v17.0.1). With nestedRows: true, calling hot.setSourceDataAtCell(1, 'artist', 'UPDATED') should update the first child row (visual row 1 = Child of Parent A). Instead, the old DataSource.setAtCell() wrote directly to this.data[row] — which is the second top-level entry in the flat array (Second Parent). The child row is left unchanged.
How to observe the bug:
  1. The grid has: Parent A → Child (row 1), then Second Parent (row 2).
  2. Click "Call setSourceDataAtCell(1, 'artist', 'UPDATED')" — this targets visual row 1 (the Child).
  3. In the "After" data panel, see that Second Parent's artist changed to UPDATED. The child is unchanged.
  4. Compare with the PR Fix page where the Child is correctly updated.
