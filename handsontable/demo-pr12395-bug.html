PR #12395 - dateFormat TypeScript type for intl-date cell type

Build: Released v17.0.1 BUG PR #12395 Fix FIXED
BUG present (v17.0.1 TypeScript types). The dateFormat option in settings.d.ts is typed as string only. The intl-date cell type passes dateFormat directly into new Intl.DateTimeFormat(locale, dateFormat), so it fully supports Intl.DateTimeFormatOptions objects at runtime — but TypeScript rejects the object form with TS2322.
Note: This is a TypeScript types-only fix. Runtime behavior is not affected. The grid below uses plain JavaScript — the object form of dateFormat already works at runtime in v17.0.1. The bug is visible only in TypeScript projects.
// TypeScript error in v17.0.1 when using Intl.DateTimeFormatOptions object: src/MyGrid.tsx:12:7 error TS2322: Type '{ year: string; month: string; day: string; }' is not assignable to type 'string'. 12 │ dateFormat: { year: 'numeric', month: 'long', day: '2-digit' }, │ ~~~~~~~~~~ // The type declaration in settings.d.ts (v17.0.1): dateFormat?: string; // ← too narrow; Intl.DateTimeFormatOptions not accepted 
// BUG — handsontable/types/settings.d.ts (v17.0.1):
interface GridSettings {
  ...
  dateFormat?: string;
  // Intl.DateTimeFormatOptions is NOT in the union → TS error TS2322
}
How to observe the bug:
  1. In a TypeScript project with handsontable@17.0.1, configure an intl-date column with an object dateFormat (see snippet above).
  2. BUG: tsc reports TS2322 — the object form is rejected even though it works at runtime.
  3. Workaround (v17.0.1): cast with dateFormat: {'{'} ... {'}'} as any or as string.
  4. The demo grid below uses plain JavaScript — the object form works fine at runtime. ✓
  5. Compare with the Fix page where the type is widened.
