PR #12340 - DataProvider error toasts via Notification plugin

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (released v17.0.1). In v17.0.1 neither dataProvider nor notification options exist. When a fetch fails, the developer is responsible for all error handling manually. There is no built-in error toast or Refetch button - only a raw alert() can be shown. The "Refetch" button below does nothing useful because there is no dataProvider plugin to call.

Steps to reproduce:
  1. Click "Trigger fetch error" below.
  2. A plain browser alert() appears - this is the only built-in error mechanism available.
  3. Click "Refetch" - nothing happens. There is no plugin to retry the fetch.
  4. Switch to the PR Fix build to see the notification toast with a proper Refetch action.
