PR #12121 - Stack overflow when pasting 50k+ rows

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (v17.0.1). htmlToGridSettings() in parseTable.js uses the spread operator to push all tbody rows at once: sections.push(...Array.from(section.rows)). When the table has 50,000+ rows, JavaScript exceeds the maximum call stack size (each element becomes a function argument), throwing RangeError: Maximum call stack size exceeded and crashing the paste operation. 
// parseTable.js — BEFORE (buggy):
...Array.from(checkElement.tBodies).reduce((sections, section) => {
  sections.push(...Array.from(section.rows));  // ← crashes with 50k+ rows
  return sections;
}, []),
Drag the slider to set row count, then click Run test. The crash threshold varies by browser — typically between 100,000–500,000 arguments. At 50,000 rows it reliably crashes in Chrome/Firefox.
