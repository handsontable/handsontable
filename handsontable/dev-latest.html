Undo after column remove with merged cells

Version: Released v17.1.0 — BUG PR Build fixed
Bug: Removing a column that overlaps a merged cell area and then pressing Ctrl+Z restores the column data but leaves the cells unmerged.

How to reproduce:
1. The grid loads with several merge configurations (see labels below).
2. Right-click a highlighted column header and choose Remove column — or use the buttons.
3. Press Ctrl+Z (or click Undo).
4. BUG: the merged cell is not restored — cells remain separate.
5. Switch to the PR Build tab to see the fix.
