PR #12305 - Shared HyperFormula engine with autoColumnSize/autoRowSize no longer causes TypeError

Build: Released v17.0.1 BUG PR #12305 Fix FIXED
FIX applied (PR #12305 — local build). Both AutoColumnSize#onAfterFormulasValuesUpdate and AutoRowSize#onAfterFormulasValuesUpdate now filter the incoming changes array to only process entries where change.address.sheet === sheetId. Changes belonging to other sheets are skipped before any index translation, preventing toVisualRow / toVisualColumn from receiving out-of-range indices. 
// autoColumnSize.js / autoRowSize.js — onAfterFormulasValuesUpdate:
const formulasPlugin = this.hot.getPlugin('formulas');
const sheetId = formulasPlugin?.sheetId;

const changedColumns = changes.reduce((acc, change) => {
  if (sheetId !== null && sheetId !== undefined && change.address?.sheet !== sheetId) {
    return acc; // skip cross-sheet changes
  }
  // ... rest of handler
});
