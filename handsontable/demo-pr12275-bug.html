PR #12275 - NestedHeaders ghost table width regression (collapse/expand jumping)

Build: v17.0.1 + regression CSS BUG PR Fix FIXED
Regression simulated. PR #12129 (rowspan in NestedHeaders) changed the ghost table from HTML-string rendering to a DOM-based approach that omitted the <thead> wrapper. This broke .htGhostTable table thead th { border-bottom-width: 0 } — the theme's bottom-border suppression for ghost headers. Without it, each ghost header cell gets an extra bottom border, skewing the column width measurements returned on every collapse/expand cycle. The regression CSS is injected via <style id="regression-style"> in this page.
/* Injected to simulate the regression */
.htGhostTable table thead th {
  border-bottom-width: 4px !important;
}
How to observe the bug:
  1. Click "Snapshot initial widths" to record column widths now.
  2. Click "Collapse A–C" to collapse the "Personal Info" group.
  3. Click "Expand A–C" to expand it back.
  4. Repeat steps 2–3 several times. Watch the width table — columns should drift wider with each cycle (highlighted in red).
  5. Compare with the PR Fix page where widths stay stable.
