PR #12235 - Fast scrollbar drag rendering (StickyScrollStrategy)

Build: Released v17.0.1 BUG PR Fix FIXED
No sticky-scroll strategy (v17.0.1). During fast native scrollbar drag, the spreader element stays at position: relative — JavaScript must reposition it on every scroll event. At high scroll speed the browser paints frames before JS finishes, causing blank/white flicker or a visible position "jump" on mouseup.
How to observe the issue:
  1. Click and hold the vertical scrollbar thumb on the right side of the grid below.
  2. Drag it rapidly up and down several times.
  3. Release the mouse.
  4. Look for: white/blank rows during drag, or a visible position jump on release.
  5. Watch the Sticky Mode indicator — it will never turn green (no sticky strategy).
  6. Compare with the PR Fix page where sticky mode activates during drag.
