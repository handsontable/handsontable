PR #12477 - clear() leaves all cells selected

Build: Released v17.0.1 BUG PR #12477 Fix FIXED
BUG present (v17.0.1). Calling hot.clear() clears all cell data correctly, but leaves every cell in a selected state. Internally it calls selectAll() to collect cells for emptying but never calls deselectCell() afterwards, so the full-table selection persists as a visible side-effect. 
// BUG — core.js clear() before fix:
this.clear = function() {
  this.selectAll();
  this.emptySelectedCells();
  // ← missing: this.deselectCell()
  // all cells remain selected after this returns
};
How to observe the bug:
  1. The grid starts with some data and a cell selected at (0, 0).
  2. Click "hot.clear()".
  3. BUG: all cells are highlighted / selected after clearing — getSelected() is not undefined.
  4. Compare with the Fix page.
