Build: Released v17.0.1 BUG PR #12254 Fix FIXED PR #12255 Fix FIXED
FIX applied (PR #12255 — local build). Same root cause as PR #12254: isEmptyRow/isEmptyCol in metaSchema.js now compare non-empty cell values against their corresponding dataSchema default. When an explicit dataSchema is configured and a cell's value equals the schema default for that column (e.g. false === false), the cell is considered empty. Without an explicit dataSchema (duck-schema), existing behavior is preserved. 
// metaSchema.js — isEmptyRow updated to use schema defaults:
const schema = this.getSchema();
// Per-cell: compare against schema default for that property
if (schema && Object.prototype.hasOwnProperty.call(schema, prop)) {
  const schemaDefault = schema[prop];
  if (value === schemaDefault) continue; // treat as empty
}
