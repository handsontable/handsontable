PR #12118 - manualColumnResize / manualRowResize broken in CSS-scaled container

Build: Released v17.0.1 BUG PR Fix FIXED
FIX applied (PR #12118 — local build). New utility functions getElementScaleFactor() and normalizeVisualDelta() detect the CSS scale applied to the container and divide the drag delta by it. The handle is positioned using outerWidth()/outerHeight() (layout size) instead of getBoundingClientRect() (visual size), so it aligns correctly at any scale. 
// manualResize/utils.js (new in PR #12118):
export function getElementScaleFactor(element, axis = 'horizontal') {
  const transformedSize = axis === 'vertical' ? element.getBoundingClientRect().height : element.getBoundingClientRect().width;
  const unscaledSize    = axis === 'vertical' ? element.offsetHeight : element.offsetWidth;
  return (transformedSize / unscaledSize) || 1;  ← e.g. 0.5 for scale(0.5)
}

// In #onMouseMove:
const change = normalizeVisualDelta(event.pageX - this.startX, this.#horizontalScaleFactor);
// normalizeVisualDelta divides by scale: 25 visual px / 0.5 = 50 layout px ✓
