PR #12151 - afterScrollVertically/afterScrollHorizontally fire without position change

Build: Released v17.0.1 BUG PR Fix FIXED
FIX applied (PR #12151 — local build). Overlays now caches the last scroll position for each axis in #lastVerticalScrollPositionForCallback and #lastHorizontalScrollPositionForCallback. Before invoking onScroll() (which fires the hook), #didVerticalScrollPositionChange() compares the current position to the cached one. If unchanged, the hook is suppressed and the cache is not updated. 
// overlays.js — AFTER (fixed):
if (this.verticalScrolling && this.#didVerticalScrollPositionChange()) {
  this.inlineStartOverlay.onScroll(); // fires afterScrollVertically
}
if (this.horizontalScrolling && this.#didHorizontalScrollPositionChange()) {
  this.topOverlay.onScroll(); // fires afterScrollHorizontally
}

// #didVerticalScrollPositionChange() caches and compares getScrollPosition()
// → returns false (suppresses hook) if position is same as last time
