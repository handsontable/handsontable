PR #12151 - afterScrollVertically/afterScrollHorizontally fire without position change

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (v17.0.1). In Overlays#refreshAll(), the afterScrollVertically and afterScrollHorizontally hooks fire whenever the verticalScrolling / horizontalScrolling flags are true — regardless of whether getScrollPosition() returned a different value. Click "Run duplicate-scroll test" to trigger two back-to-back refreshAll() calls at the same position. The hook fires twice instead of once.
How to observe the bug:
  1. Click "Run duplicate-scroll test". This simulates a Walkontable scenario where verticalScrolling = true is set twice but the scroll position does not change.
  2. Observe the afterScrollVertically total counter: it reads 2. It should be 1 — position did not change on the second call.
  3. Click "Run horizontal test" for the same issue on the horizontal axis.
  4. Compare with the PR Fix page where both counters stay at 1.
