PR #12247 - currentRowClassName / currentColClassName update dynamically via updateSettings

Build: Released v17.0.1 BUG PR #12247 Fix FIXED
FIX applied (PR #12247 — local build). selection.updateHighlightClassNames() is now called inside updateSettings() (in core.js, just before the afterUpdateSettings hook). It propagates the new class names to all already-cached highlight layer objects by updating both this.options and each cached highlight's settings.className. 
// core.js — updateSettings(), just before afterUpdateSettings hook:
selection.updateHighlightClassNames();
instance.runHooks('afterUpdateSettings', settings);

// selection.js — new method:
updateHighlightClassNames() {
  this.highlight.updateHighlightClassNames({
    rowClassName: this.settings.currentRowClassName,
    columnClassName: this.settings.currentColClassName,
    // ...
  });
}
