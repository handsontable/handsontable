PR #12247 - currentRowClassName / currentColClassName ignored by updateSettings

Build: Released v17.0.1 BUG PR #12247 Fix FIXED
BUG present (v17.0.1). updateSettings({ currentRowClassName: 'newClass' }) updates the setting value but does not propagate it to already-cached highlight objects. The highlight system continues applying the old class name until the page is reloaded, so the row/column highlight color does not change visually.
How to observe the bug:
  1. The grid starts with currentRowClassName: 'row-old' (red background). Click any cell — the row turns red.
  2. Click "Change to green via updateSettings".
  3. Click the same cell again.
  4. BUG: the row stays red — old class is still applied. Cells with row-new (green) = 0.
  5. Compare with the Fix page where the row turns green immediately.
