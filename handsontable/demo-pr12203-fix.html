PR #12203 - Autofill no longer overwrites extra visible columns across hidden columns

Build: Released v17.0.1 BUG PR #12203 Fix FIXED
FIX applied (PR #12203 — local build). When source is 'Autofill.fill', getInputValue() now uses the raw column/row index (c / r) rather than the visual index computed after skipping hidden rows/columns. This prevents the input pattern from repeating into visible columns beyond the autofill drag target. 
// core.js — populateFromArray inner loop (simplified):
const isAutofillSource = source === 'Autofill.fill' || source === 'autofill.fill';
const sourceRow = isAutofillSource ? r : visualRow;
const sourceColumn = isAutofillSource ? c : visualColumn;
let value = getInputValue(visualRow, visualColumn);
let value = getInputValue(sourceRow, sourceColumn);
