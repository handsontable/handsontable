PR #12150 - Filter searchMode:apply input lag (hideRow called N times per keystroke)

Build: Released v17.0.1 BUG PR Fix FIXED
BUG present (released v17.0.1). With filters: { searchMode: 'apply' }, typing in the filter search box calls hiddenRows.hideRow(index) individually for every non-matching row — O(N) hook invocations per keystroke. With large datasets this causes noticeable input lag.

Open the filter dropdown on the City column, click in the search box, and type any letter. Watch the counters update — beforeHideRows is called once per non-matching row instead of once total.
How to test: 1. Click ▼ on the City column header to open the filter. 2. Click in the Search input. 3. Type any letter (e.g. a, n, s). 4. Watch the hook counters — beforeHideRows fires N times per keystroke (N = non-matching rows).
