PR #12115 - Comment editor correctly positioned at merged cell right border

Build: Released v17.0.1 BUG PR #12115 Fix FIXED
FIX applied (PR #12115 — local build). A new getEditorAnchorWidth(metaColspan, TD, renderableColumnWidth) helper reads the merged cell's actual rendered width via outerWidth(TD) when metaColspan > 1, replacing the single-column width. A mergedBorderCompensation of 1px is also subtracted to align the editor exactly at the merged cell's right border. 
// comments.js — showAtCell() anchor width:
const lastColWidth = isBeforeRenderedColumns ? 0 : wtTable.getColumnWidth(renderableColumn);
const lastColWidth = isBeforeRenderedColumns ? 0 :
  getEditorAnchorWidth(metaColspan, TD, wtTable.getColumnWidth(renderableColumn));
const mergedBorderCompensation = metaColspan > 1 ? 1 : 0;
let x = left + rootWindow.scrollX + lastColWidth - mergedBorderCompensation;
