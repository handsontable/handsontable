PR #12290 - Space key in filter search clears the list

Build: Released v17.0.1 BUG PR Fix FIXED
FIX applied (PR #12290 patch). The search input value is now trimmed with .trim() before being passed to the filter logic. Leading/trailing spaces are stripped, so pressing Space has no effect on results. Whitespace-only input shows the full unfiltered list.

Steps to verify:
  1. Click the dropdown arrow ▼ on the City column header.
  2. Click inside the Search input field.
  3. Press Space — all cities remain visible (whitespace = empty query).
  4. Type New then press Space — "New York" and "New Orleans" stay in the list.
  5. Continue typing New York — only "New York" remains.
Patch applied at runtime via a capture-phase input listener that temporarily sets the trimmed value on the input element before the plugin's bubble-phase handler reads it — equivalent to the one-liner .trim() fix in multipleSelect.js (PR #12290).
