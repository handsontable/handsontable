Build: Released v17.0.1 BUG PR #12153 Fix FIXED
FIX applied (PR #12153 — local build). Checkbox cells in a multi-layer mixed selection now correctly receive false (uncheckedTemplate) on Delete — not null and not the noValue class. All checkbox + non-checkbox changes are batched into a single setDataAtCell call, producing exactly one undo action that fully restores all data on Ctrl+Z. Also fixed: emptySelectedCells no longer registers header coordinates (negative row/col) in undo actions when Ctrl+A-like selection is used with row/col headers. 
// checkboxRenderer.js — batch all uncheck changes into one setDataAtCell call:
if (uncheckCheckbox) {
  const allChanges = changesChunks.flat();
  if (allChanges.length > 0) hotInstance.setDataAtCell(allChanges);
} else {
  changesChunks.forEach(chunk => { if (chunk.length > 0) hotInstance.setDataAtCell(chunk); });
}

// core.js — guard emptySelectedCells against header coordinates:
const fromRow = Math.max(topStart.row, 0);
const toRow = Math.min(bottomEnd.row, this.countRows() - 1);
if (fromRow > toRow || fromColumn > toColumn) return;
