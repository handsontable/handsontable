PR #12631 - Missing res.ok check causes SyntaxError

Build: Released v17.0.1 BUG PR #12631 Fix FIXED
BUG (docs example — all 4 variants: JS, TS, React, Angular). The server-side-spring recipe examples call res.json() directly without first checking res.ok. When the backend is unreachable (404, 500, network error), the server returns an HTML error page. Calling res.json() on HTML throws:
SyntaxError: Unexpected token '<', "<!DOCTYPE..."
instead of a meaningful HTTP error, making the failure invisible and hard to debug. 
// BUG — all 4 variants (JS, TS, React, Angular) had this pattern:
async function fetchRows({ page, pageSize, sort }) {
  const res = await fetch(`/api/cars?page=${page}&pageSize=${pageSize}`);
  // ← no res.ok check here!
  const json = await res.json(); // throws SyntaxError when server returns HTML error page
  return { data: json.data, meta: { total: json.total } };
}
How to observe the bug:
  1. Click "Simulate HTTP 404 (no res.ok check)".
  2. BUG: The error thrown is SyntaxError: Unexpected token '<' — cryptic, not actionable.
  3. Compare with the Fix page which shows Error: HTTP 404.
// javascript/example1.js — BUG (missing res.ok check)
async function fetchRows({ startRow, amount, sortBy, sortOrder }) {
  const url = `/api/cars?from=${startRow}&amount=${amount}` +
    (sortBy ? `&sortBy=${sortBy}&sortOrder=${sortOrder}` : '');
  const res = await fetch(url);
  // missing: if (!res.ok) throw new Error(`HTTP ${res.status}`);
  const json = await res.json(); // SyntaxError if server returns HTML
  return { data: json.data, meta: { total: json.total } };
}
