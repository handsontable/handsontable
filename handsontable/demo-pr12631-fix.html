PR #12631 - res.ok check throws clear HTTP error

Build: Released v17.0.1 BUG PR #12631 Fix FIXED
FIX applied (PR #12631). All 4 variants of the server-side-spring recipe (JS, TS, React, Angular) now check res.ok before calling res.json(). When the server returns a non-2xx response, a clear Error: HTTP 404 is thrown instead of a cryptic SyntaxError. This surfaces the real problem - the server is down or returning an error - making debugging straightforward. 
// FIX — one-line guard added to all 4 variants after each fetch() call:
async function fetchRows({ startRow, amount, sortBy, sortOrder }) {
  const res = await fetch(`/api/cars?from=${startRow}&amount=${amount}`);
  if (!res.ok) throw new Error(`HTTP ${res.status}`);  // ← NEW: clear HTTP error
  const json = await res.json();  // only called when res.ok is true
  return { data: json.data, meta: { total: json.total } };
}
