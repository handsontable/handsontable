PR #12491 - Password cell type — hashRevealDelay option

Build: Released v17.0.1 BEFORE PR #12491 Fix AFTER
BEFORE (v17.0.1). The type: 'password' cell type uses a native <input type="password"> which hides each character immediately as you type. There is no way to briefly show the typed character before masking it — a UX pattern common in mobile keyboards and password fields. The hashRevealDelay option does not exist yet.
  • Every character is hidden the instant you press a key.
  • No feedback — you cannot tell if you mistyped.
  • hashRevealDelay passed as a setting is silently ignored.
// BEFORE — password editor (v17.0.1):
// PasswordEditor always uses <input type="password"> — instant masking:
createElements() {
  super.createElements();
  this.TEXTAREA.type = 'password';  // set once, never switched to "text"
  this.TEXTAREA.className = 'handsontableInput';
}
// No hashRevealDelay, no #realValue, no #revealTimer — option is ignored entirely.
How to observe the behavior:
  1. Double-click any cell in the "Password" column to open the editor.
  2. Start typing — every character is immediately hidden as a bullet (●).
  3. Press Enter. The stored value is masked with * as expected.
  4. Click "Read stored values" to see getSourceDataAtCell() returns the real string.
  5. Compare with the Fix page where hashRevealDelay: 1000 keeps each character visible for 1 second.
