PR #12344 - Drag-select during auto-scroll (DragToScroll)

Build: Released v17.0.1 BUG PR #12344 Fix FIXED
BUG present (v17.0.1). When you hold the left mouse button and drag past the viewport edge, the grid auto-scrolls to reveal new cells — but the selection range freezes at the last visible edge cell. The viewport scrolls away under the cursor, but the selection never extends into the newly revealed area.
  • Auto-scroll fires — the grid scrolls correctly. ✓
  • Selection stops at the edge cell visible when dragging began. ✗
  • Newly revealed rows/columns are never included in the selection. ✗
// BUG — DragToScroll plugin (v17.0.1):
// Auto-scroll tick fires but only scrolls — no selection update:
runVertically(row) {
  this.hot.scrollViewportTo({ row, col: this.hot.getSelectedRangeLast()?.to.col });
  // selection.setRangeEnd() is NEVER called — selection stays frozen
}
runHorizontally(col) {
  this.hot.scrollViewportTo({ row: this.hot.getSelectedRangeLast()?.to.row, col });
  // same: scroll happens, selection does not follow
}
How to observe the bug:
  1. Click a cell near the middle of the grid (e.g., row 10, column D).
  2. Hold the left mouse button and slowly drag downward past the bottom edge of the grid.
  3. BUG: the grid scrolls, but the selection range freezes — newly revealed rows are not highlighted.
  4. Release and check the "Last selection" counter below — it stops at the edge row.
  5. Compare with the Fix page where the selection tracks the scroll.
