FIX applied (PR #12490). The dragToScroll TypeScript type now correctly reflects the full API: boolean | { interval?: { min?: number; max?: number }; rampDistance?: number }. All object-form configurations that were always valid at runtime now compile without error.
This page loads dist/handsontable.full.min.js built from the develop branch which includes PR #12490. The fix is a one-line change in dragToScroll.d.ts — no runtime code was modified.
Type declaration (after fix)
// dragToScroll.d.ts — AFTER fix
export type Settings = boolean | {
  interval?: { min?: number; max?: number };
  rampDistance?: number;
};
All forms now valid TypeScript ✓
// boolean — unchanged, still valid
dragToScroll: true

// full object
dragToScroll: { interval: { min: 20, max: 500 }, rampDistance: 120 }

// partial — only interval.min
dragToScroll: { interval: { min: 50 } }

// partial — only rampDistance
dragToScroll: { rampDistance: 80 }
TypeScript — no errors (after fix) ✓
All four configurations above compile cleanly. No as any cast required.
