BUG present (v17.0.1). isEmptyRow only treats null, '', and undefined as empty. With dataSchema: { name: '', active: false }, spare rows have active: false — which is the schema default but fails the empty check. minSpareRows always sees 0 spare rows and keeps adding more on every render. The demo stops at 12 rows to prevent a browser hang.
How to observe the bug:
  1. The grid initializes normally (no dataSchema or minSpareRows yet).
  2. Click "Apply dataSchema + minSpareRows: 1 (trigger bug)".
  3. Watch the row count shoot up past 3 — stopped by a safeguard at 12. In a real app the tab would crash.
  4. Compare with the PR #12254 Fix page where the count stays at 3.
