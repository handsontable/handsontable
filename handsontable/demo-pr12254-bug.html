PR #12254/#12255 - dataSchema non-null defaults cause minSpareRows to grow rows infinitely

Build: Released v17.0.1 BUG PR #12254 Fix FIXED PR #12255 Fix FIXED
BUG present (v17.0.1). isEmptyRow only treats null, '', and undefined as empty. With dataSchema: { name: '', active: false }, newly-added spare rows have active: false — which is the schema default but fails the empty check. minSpareRows always sees 0 spare rows and keeps appending rows on every render. The demo stops automatically at 12 rows to prevent a browser hang.
How to observe the bug:
  1. Click "Enable minSpareRows: 1 and trigger".
  2. Watch the row count shoot up to 12 (stopped by a safeguard). In a real app this would grow until the tab crashes.
  3. The afterCreateRow log shows multiple events fired in a single render cycle.
  4. Compare with the PR #12254 Fix page where the count stabilizes at 3.
