PR #12254/#12255 - dataSchema non-null defaults cause minSpareRows to grow rows infinitely

Build: Released v17.0.1 BUG PR #12254 Fix FIXED PR #12255 Fix FIXED
BUG present (v17.0.1). The isEmptyRow implementation treats only null, '', and undefined as empty. When dataSchema: { name: '', active: false } is used with a checkbox column, newly-added spare rows have active: false — which is the schema default but is not treated as empty. So minSpareRows always sees 0 spare rows and keeps appending new rows on every user interaction.
How to observe the bug:
  1. Click "Add a data row via API" to trigger a change — watch the row count increase by more than 1 (spare row cascade).
  2. Or click directly in the Active checkbox in the grid — each click adds extra rows.
  3. Watch the Rows added by afterCreateRow counter — it keeps climbing.
  4. Compare with the PR #12254 Fix or PR #12255 Fix page where the count stabilizes.
