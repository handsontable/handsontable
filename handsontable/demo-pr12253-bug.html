PR #12253 - Dropdown button icon invisible on active + filtered column header

Build: Released v17.0.1 BUG PR #12253 Fix FIXED
BUG present (v17.0.1). When a column has both an active filter (htFiltersActive) and is the currently selected column (ht__active_highlight), a CSS rule overrides the dropdown .changeType::after icon color with var(--ht-header-active-foreground-color) (white in the main theme). The icon becomes invisible against the light button background.
How to observe the bug:
  1. Click "Apply filter + activate column A" to apply a filter on column A and select a cell in it.
  2. Look at the column A header — it has both the filter badge (orange/yellow tint) and the active highlight.
  3. The ▾ dropdown button icon is invisible (white on light background).
  4. The metric below shows the computed icon color vs the button background color.
  5. Compare with the Fix page where the icon is visible.
