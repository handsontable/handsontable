PR #12340 - DataProvider error toasts via Notification plugin

Build: Released v17.0.1 BUG PR Fix FIXED
FIX applied (PR #12340). With notification: true set alongside dataProvider, failed fetchRows calls now show a persistent error toast (duration: 0) with a Refetch action button. Clicking Refetch calls fetchData() again. The old Dialog-based error UI has been removed.

Steps to verify:
  1. Click "Trigger fetch error" - this sets a failure flag and calls fetchData().
  2. After ~400 ms the fetch rejects. A persistent error toast appears at the bottom of the viewport.
  3. The toast contains the error message and a Refetch button.
  4. Click Refetch in the toast - the failure flag resets, the retry succeeds, and the grid loads data.
  5. The toast dismisses automatically once the retry succeeds.
