PR #12278 — Wrappers skip init-only settings in updateSettings

Demo: Bug v17.0.1 Fix local build
Bug — v17.0.1 and core Handsontable: Passing an init-only setting (e.g. renderAllRows) to updateSettings() throws an error. Because the whole call is aborted by the throw, no setting in that call is applied — even a perfectly valid one like colHeaders: false. In React/Vue/Angular wrappers this crash bubbled up whenever the user changed a prop that maps to an init-only setting.
