DEV-1824 — setDataAtCell TypeScript signature fix

State: BEFORE fix v17.1.0 CDN AFTER fix fixed
Bug (TypeScript compile error): In v17.1.0, the setDataAtCell type signature defined column as number | null only. When calling setDataAtCell(changes, source) in the array form, TypeScript raised:

TS2345: Argument of type 'string' is not assignable to parameter of type 'number'

Developers had to use (hot as any).setDataAtCell(...) as a workaround. The runtime behavior was always correct — only the TypeScript type was wrong.

TypeScript code — BEFORE fix BROKEN TS

// ❌ TypeScript ERROR before the fix: // TS2345: Argument of type 'string' is not assignable // to parameter of type 'number' hot.setDataAtCell([[0, 0, 'new value']], 'mySource'); // ✅ Workaround required (loses type safety): (hot as any).setDataAtCell([[0, 0, 'new value']], 'mySource'); // Same issue for setDataAtRowProp: hot.setDataAtRowProp([[0, 'prop', 'value']], 'mySource'); (hot as any).setDataAtRowProp([[0, 'prop', 'value']], 'mySource'); // The old (broken) type signature was: // setDataAtCell(row: number | unknown[][], column?: number | null, ...) // ^^^^^^^^^^^^ // 'string' not assignable to 'number | null'
