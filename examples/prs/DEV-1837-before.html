DEV-1837: Before fix – noImplicitThis: false (Handsontable 14.6.1)

State before the fix: tsconfig.json had "noImplicitThis": false, which meant TypeScript did not enforce explicit this typing in function expressions. The source used this: any annotations (or none at all) in core.ts methods like suspendRender, validateCells, and in all built-in renderers. This demo uses Handsontable 14.6.1 — a version before the fix — to show the same runtime behavior. No functional difference is visible at runtime; the fix is purely a TypeScript strictness improvement.
