DEV-1770 — Excel paste with sanitizer BUG — v17.0.1

CopyPaste: pasting a single Excel cell erases the cell directly below the paste target when a custom sanitizer is used.

Version: Before fix v17.0.1 After fix v17.1.0
Scenario: A custom sanitizer strips HTML to plain text (as shown in the v17 migration guide). When a single cell is copied from Excel, Excel puts text/plain = "value\r\n" (trailing CRLF). The sanitizer removes the <table> HTML, so the paste falls back to text/plain. SheetClip reads the trailing \r\n as a row separator and emits an extra empty row, which overwrites the cell below the target.

Steps:
1. Click "Select B4".
2. Click "Simulate Excel paste".
3. Expected: B4 = "PASTED", B5 = "B5" (unchanged).
4. Actual (bug): B4 = "PASTED", B5 = "" (erased).
